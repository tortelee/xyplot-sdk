## 🎯 一句话总结

> 一个**零 UI 依赖的 C++ 嵌入式绘图 SDK**，通过两个抽象接口（`IRenderDevice` + `IInputSource`）与客户任意显示/交互系统对接。本周交付 MVP（折线图 + 散点图 + 多 Y 轴），1 周完成。

---

## 📋 PO 反馈要点 & 架构响应

| PO 反馈 | 架构响应 |
|---------|---------|
| **Q4**: "交互库不可控，客户显示系统未知" | ❗ **核心引擎零 UI 依赖**。定义 `IRenderDevice`（~20 个图元方法）+ `IInputSource`（统一事件），客户实现接口对接自己的框架 |
| **Q2**: 产品形态 = SDK/库 | API 设计优先。`Plot::render(device)` 是唯一渲染入口 |
| **Q1**: 图范围 = 广义 | 架构预留 3D 扩展（后续版本增加 `drawTriangle()` 等） |
| **Q5**: 实时 + 大数据 | 坐标变换纯计算，不分配 GPU 资源；数据 API 支持就地更新 |
| **Q9**: 1 周 | P0 范围压缩为 Line + Scatter + Multi-Y-Axis |
