## 二、需求拆解与分析

### 2.1 "XY Plot" 的边界定义

"XY Plot" 在不同语境下有不同的含义。需要 PO 确认范围：

**狭义 XY Plot**：二维笛卡尔坐标系下的图线（line plot, scatter plot）
**广义 XY Plot（常见于仿真领域）**：包括但不限于：

```
├── 二维图 (2D Plots)
│   ├── 折线图 (Line Plot)
│   ├── 散点图 (Scatter Plot)
│   ├── 阶梯图 (Step/Stair Plot)
│   ├── 条形图/柱状图 (Bar/Column Chart)
│   ├── 面积图 (Area Plot)
│   ├── 误差棒图 (Error Bar Plot)
│   ├── 瀑布图 (Waterfall Plot)
│   ├── 极坐标图 (Polar Plot)
│   ├── 史密斯圆图 (Smith Chart)
│   ├── 等高线图 (Contour Plot)
│   ├── 热力图 (Heatmap)
│   ├── 矢量场图 (Vector/Quiver Plot)
│   ├── 流线图 (Streamline Plot)
│   └── 直方图 (Histogram)
│
├── 三维图 (3D Plots)
│   ├── 3D 曲面图 (Surface Plot)
│   ├── 3D 散点图 (3D Scatter)
│   ├── 3D 线图 (3D Line/Parametric)
│   ├── 网格图 (Mesh/Wireframe Plot)
│   ├── 体积渲染 (Volume Rendering)
│   └── 瀑布图 3D (3D Waterfall)
│
├── 特殊图类型
│   ├── 波特图 (Bode Plot) — 频域分析
│   ├── 奈奎斯特图 (Nyquist Plot) — 控制理论
│   ├── 眼图 (Eye Diagram) — 信号完整性
│   ├── 星座图 (Constellation Diagram) — 通信
│   ├── 伯德图/根轨迹 (Root Locus)
│   └── 频谱图/瀑布图 (Spectrogram)
│
└── 复合/多轴图
    ├── 双Y轴/多Y轴图
    ├── 子图布局 (Subplot/Grid Layout)
    └── 叠加图 (Overlay)
```

## 四、关键架构推导

### 从 PO 回复推导出的核心约束：

```
约束 1: 产品是 SDK（非独立 App）
  → API 设计是最高优先级
  → 必须对调用方友好，最小化依赖

约束 2: 显示/交互库不可控
  → 核心引擎不能依赖 Qt/VTK/任何特定 UI 框架
  → 必须定义抽象渲染接口 (IRenderDevice)
  → 必须定义抽象输入接口 (IInputSource)
  → 客户实现这些接口对接自己的显示系统

约束 3: 实时 + 大数据
  → 渲染路径必须高效
  → 数据管理必须考虑内存和缓存策略

约束 4: 1 周时间线
  → 仅 P0 图类型：Line、Scatter、Multi-Y-Axis
  → 架构打底，后续图类型以插件方式扩展
  → 参考后端最小化
```
